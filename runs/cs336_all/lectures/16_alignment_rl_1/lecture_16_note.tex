\section{从 RLHF 的麻烦出发：为什么还需要 RLVR}

\subsection{第 16 讲先做了一次很诚实的回顾}
第 16 讲并没有直接宣告“旧方法不行了”，而是先承认 DPO 和 PPO 各自的价值。DPO 之所以流行，是因为它把一大块 RLHF 工程复杂度拿掉了：不再显式训练 reward model，也尽量避开 on-policy rollout 和 value function。PPO 则仍然是更传统、表达力更强的强化学习套路，在某些设置下依旧有效。

但课程真正想指出的是：如果你的反馈不是天然的 pairwise preference，或者你希望把可验证的正确性直接拿来做在线优化，那么单靠 DPO 这类离线偏好优化就会开始显得不够。此时，PPO 一类在线 RL 方法又会重新变得重要。

\begin{figure}[H]
\centering
\includegraphics[width=0.88\textwidth]{pdf_pages/page-05.png}
\caption{课程用 DPO 的推导结果提醒学生：它的工程吸引力来自把 RLHF 重写成更像监督学习的目标。}
\end{figure}

\begin{importantbox}{第 16 讲的起点}
这一讲不是在“推翻 DPO”，而是在问：当目标从偏好匹配走向可验证推理时，我们是否需要更直接、更在线的优化方式。
\end{importantbox}

\subsection{本章小结}
第 16 讲先承认了 DPO 的现实价值，但也把它的边界暴露出来：不是所有反馈都天然长成偏好对，也不是所有目标都适合用离线 pairwise 数据来学。

\section{DPO 的吸引力与它没法解决的问题}

\subsection{为什么 DPO 看起来像“RLHF without tears”}
课程把 DPO 概括得非常清楚：在一个非参数假设下，策略 $\pi_\theta$ 和 reward 可以建立闭式联系，于是你可以把偏好优化重写成对策略对数比值的监督学习目标。这样一来，原本 RLHF 里最让人头疼的 reward model、rollout 外循环和 critic 都能暂时退场。

从直觉上看，DPO 的更新像是在做“对好回答加正梯度、对坏回答加负梯度”，只是这个正负更新会被一个从偏好概率导出的误差项重新加权。课程在这里反复强调，它的魅力首先是工程上的，而不是哲学上的。

\subsection{为什么课程仍然对 DPO 保持谨慎}
课程同时非常明确地指出，DPO 的效果高度依赖实验设置。它在很多开源 RLHF 模型里确实好用，但 PPO 有时也能同样甚至更好地工作。更重要的是，一旦数据不是 Bradley-Terry 风格的成对比较，或者你想做真正在线的行为改进，DPO 的自然性就会下降。

\begin{knowledgebox}{课程对 DPO 的态度}
DPO 很好，但它更像是“把一类偏好优化问题做得更可训练”，而不是解决所有对齐问题的总开关。
\end{knowledgebox}

\subsection{本章小结}
DPO 的意义在于大幅降低偏好优化的门槛；它的边界在于，它仍然假设反馈能以特定的成对形式表达，而且主要是离线优化。

\section{RLHF 的副作用：过优化与模式坍缩为什么会反复出现}

\subsection{为什么 RLHF 难以无限 scale out}
课程在这一段的语气明显变得更谨慎。只要你在有限偏好信号上持续做优化，就很容易把模型推到 reward 过拟合区间。也就是说，reward 确实会上升，但真实目标不一定同步提升，甚至可能开始变差。这正是 overoptimization 的经典结构。

课程给出的洞察很重要：偏好奖励并不是“真理”，它只是某个噪声很大、覆盖有限的近似目标。所以在这个目标上过度优化，几乎必然会出现过拟合。对于 RLHF 来说，这不是实现 bug，而是问题设定内生的风险。

\subsection{模式坍缩说明模型不再像概率模型那样工作}
课程用 calibration 和 entropy 分布的变化说明 mode collapse 的含义。经过 RLHF 之后，模型往往不再像一个良好校准的概率模型，而更像一个被特定偏好信号强行拉扯过的决策器。它可能更“会赢 reward”，却更不多样、也更不可信。

\begin{figure}[H]
\centering
\includegraphics[width=0.88\textwidth]{pdf_pages/page-13.png}
\caption{课程把模式坍缩和校准恶化并列展示，提醒学生 RLHF 会改变模型的概率结构。}
\end{figure}

\begin{warningbox}{为什么这件事对 reasoning 很关键}
如果你想让模型在数学、代码或长链推理上持续探索，模式坍缩会直接伤害这种探索能力。模型越早收缩到几种“看起来最容易赢”的响应模式，就越难继续从奖励中学出更强推理。
\end{warningbox}

\subsection{本章小结}
第 16 讲在这里做了一个非常关键的转向：与其在一个有噪声的偏好信号上无休止优化，不如寻找那些更可验证、更难被奖励黑客化的目标。

\section{为什么这讲会把课程从 GPT-3.5 直接推进到 o1 / r1}

\subsection{课程主线在这里发生了明显转向}
官方材料在这里给了一页很直白的过渡：`pre-training + RLHF` 大致把行业推到了 GPT-3.5 / ChatGPT 这一代，而这讲想讨论的，是为什么后续的 reasoning 模型会明显更依赖 RL 和 test-time compute，而不是只靠更大的预训练。

\begin{figure}[H]
\centering
\includegraphics[width=0.82\textwidth]{pdf_pages/page-14.png}
\caption{课程明确把这一讲定位成“从 GPT-3.5 时代推进到 o1 / r1 时代”的转折点。}
\end{figure}

\subsection{RLVR 的目标不是再做一遍 RLHF，而是扩大 RL 的适用域}
课程紧接着又补了一页非常关键的动机图：问题不在于 RLHF 没有价值，而在于它在偏好信号上难以 cleanly scale out。一旦过优化开始变成主导风险，继续在 noisy preference reward 上堆更多 rollout，收益就会越来越差。

于是，第 16 讲真正想问的是：能不能转向那些 RL 本来就最擅长的场景，也就是奖励更接近“exactly what we want”的场景？课程用 AlphaGo、蛋白质设计和形式化数学的图示来提示学生，这类场景往往有更硬的目标函数，因此更适合被持续优化。

\subsection{本章小结}
这讲之所以重要，不是因为它又介绍了一个新缩写，而是因为它明确重设了增长路径：当偏好优化开始遇到硬上限，更可验证的奖励就成了新的扩展方向。

\section{从 policy gradient 到 PPO：为什么在线 RL 还是会回来}

\subsection{课程先把最基本的 policy gradient 写出来}
PPO 再复杂，底层仍然是 policy gradient。课程先回顾最基本的式子：
\[
\nabla_{\theta} \mathbb{E}[R] = \mathbb{E}\left[\nabla_{\theta}\log \pi_{\theta}(a\mid s)\, R(s,a)\right].
\]
其中 $s$ 是状态，$a$ 是动作，$\pi_\theta$ 是当前策略，$R(s,a)$ 是最终奖励。语言模型场景里，状态可以理解为 prompt 加已生成前缀，动作则可以是逐 token 的生成选择。

这个公式足够简单，但直接拿来做语言模型 RL 会有极高方差。于是才会有 TRPO 的信赖域思路，再到 PPO 用 clip 目标在稳定性和可优化性之间做折中。

\subsection{为什么课程还要给学生看 PPO 实现}
课程非常坦率地说，只看博客文章谈 PPO 容易产生错觉，真正难的是实现。rollout 怎么收集，KL 惩罚怎么加，advantage 怎么算，value model 怎么训练，外循环和内循环怎么配合，这些才是 PPO 在语言模型里真正昂贵的地方。

\subsection{PPO 在语言模型里到底比普通 RL 多了哪些部件}
官方 PDF 在 PPO 部分其实给了三层结构，而不只是一个 clipped objective。第一层是策略网络本身，也就是要被更新的语言模型；第二层是 reward model 和 value model，它们分别给出标量反馈和 advantage 所需的 baseline；第三层是经验缓存与 reference model，用于做 ratio、KL 和多步 inner loop 更新。

\begin{figure}[H]
\centering
\includegraphics[width=0.84\textwidth]{pdf_pages/page-21.png}
\caption{语言模型里的 PPO 不是只多一个 loss，而是 reward model、value model、reference policy、experience buffer 一起进入系统。}
\end{figure}

课程在这一页想告诉学生的是：哪怕把环境简化成“整段回答末尾给一个 dense reward”，PPO 仍然会自然长出一大套附属对象。也正因此，它在大模型里的真正代价不只是数学复杂，而是显存、推理和训练调度全都一起变重。

\begin{figure}[H]
\centering
\includegraphics[width=0.84\textwidth]{pdf_pages/page-24.png}
\caption{课程直接拿 AlpacaFarm 的 PPO 实现讲 loss 计算，强调 PPO 的复杂度主要来自工程细节。}
\end{figure}

这部分内容非常重要，因为它解释了为什么开源社区在看到 DPO 后会那么兴奋。不是大家突然不喜欢 RL 了，而是 PPO 的完整系统成本确实太高。

\subsection{reward shaping 与 GAE 是 PPO 真正难训的地方}
官方文本在 `PPO in practice` 里还点了两个经常被忽略但非常关键的实现点。

第一是 reward shaping：语言模型 RL 往往不是只在最后一个 token 上打一笔总奖，而是会沿 token 维度加上 KL 惩罚，让新策略不要离 reference 太远。官方描述得很具体：高层上是“每个 token 加 KL，末尾再加总 reward”；实践里则还会对 KL 做额外裁剪，避免 policy 突然跑飞。

第二是 generalized advantage estimate。课程特别提醒，在语言模型 RL 的很多设置里，这其实很接近一个 bandit 问题，因此会看到 \(\gamma=\lambda=1\) 这种和经典连续控制场景不太一样的设定。它的意思不是“GAE 不重要”，而是说 reward-to-go 与 value decomposition 在这里比在机器人任务里更简单，但仍然需要一个能稳定训练的 advantage 管线。

\begin{figure}[H]
\centering
\includegraphics[width=0.82\textwidth]{pdf_pages/page-23.png}
\caption{PPO outer loop 的实践图景：先收 rollout，再在固定 rollout 上做若干 inner-loop 优化。}
\end{figure}

\subsection{本章小结}
PPO 仍然是理解现代 RL 方法的参考系。课程通过回顾 policy gradient 和 PPO 实现，让学生看到“在线 RL 很强”与“在线 RL 很麻烦”这两件事为何会同时成立。

\section{GRPO：把语言模型的“同 prompt 多回答”结构直接用起来}

\subsection{GRPO 的核心思想不是更复杂，而是更简单}
课程把 GRPO 介绍成“从 PPO 出发，去掉 value function，再把 advantage 写成组内归一化奖励”。在语言模型场景里，这种做法特别自然，因为同一个 prompt 下本来就可以采样多条回答。于是每组回答天然形成一个小比较集合，你不一定要额外训练 critic 才知道哪些回答相对更好。

直观上，GRPO 的做法是：同一个 prompt 采样若干 rollout，计算每条回答的 reward，然后对组内 reward 做中心化和标准化，再用这个组内相对分数当作更新信号。课程借此说明：\textbf{语言模型的输出结构本身，就能提供一种廉价 baseline。}

\subsection{为什么课程马上开始挑 GRPO 的刺}
课程并没有把 GRPO 塑造成完美解。它明确指出，组内均值部分可以视作 baseline，但标准差归一化会引入额外偏差；同时长度归一化项还可能带来 length bias，让模型更偏向某种 CoT 长度。这说明 GRPO 的成功很大程度上来自“足够简单、足够好用”，而不是“严格无偏”。

\begin{knowledgebox}{课程对 GRPO 的真实评价}
GRPO 的魅力不是它理论上最纯，而是它在去掉 value model 之后，极大降低了 reasoning RL 的实现门槛，因此让 RLVR 快速扩散到了更多团队。
\end{knowledgebox}

\subsection{为什么课程说“stdev normalization 不是有效 baseline”}
官方材料这里有一个很值得认真保留的细节：减去组内均值这一步可以被解释成 baseline，但除以组内标准差这一步就不再是简单的 state-dependent baseline 了，它会改变梯度估计的缩放方式，因此不再保持严格无偏。

这也是课程会专门提到“unbiased-gradient version of GRPO”的原因。后来的一些修正思路，本质上是在往 `REINFORCE with leave-one-out baseline` 靠，尽量保留组结构带来的便宜优势估计，同时减少标准差归一化引入的额外偏差。

\subsection{length bias 为什么会变成 reasoning RL 里的真实问题}
官方页里还单独点出 `Length biases of GRPO`。原因很现实：一旦你对组内 reward 做标准化，并再叠加某种长度归一化或长度惩罚，模型就会开始系统性偏向某种 CoT 长度。对 reasoning 模型来说，这不是小问题，因为它会直接影响探索深度和最终答案质量。

课程在这里的态度很务实：GRPO 的成功并不意味着 objective 已经定型，相反，它说明一个足够简单的 objective 先把门打开了，接下来大家会围绕 unbiasedness、length control 和 variance scaling 再继续修。

\begin{warningbox}{为什么 GRPO 不是“公式抄下来就能复现”的方法}
只要 objective 里出现组内标准差、长度归一化和 KL 项，不同实现就可能隐含非常不同的偏置。reasoning RL 的效果，往往恰恰卡在这些看似只是常数项的地方。
\end{warningbox}

\subsection{本章小结}
GRPO 把语言模型的组结构直接变成优势估计工具。它简化了 PPO，也引入了新的偏差和长度问题，这正是后续变体不断出现的原因。

\section{为什么 RLVR 会把课程终点推到 o1 / R1 时代}

\subsection{可验证奖励让 RL 的目标第一次变得“硬”起来}
第 16 讲最重要的转向，是从 RLHF 走向 RLVR。课程的意思不是说偏好奖励没用，而是说：如果你能在数学、代码、形式逻辑或某些规则明确的任务里定义“对就是对，错就是错”的可验证奖励，那么 RL 才真正回到它最擅长的领域。

这件事的重要性在于，它把奖励从含糊的人类偏好换成了更坚硬、更抗黑客化的目标。只要奖励真能准确反映正确性，模型就更有机会通过长链推理和测试时计算获得真实能力提升，而不只是更擅长讨好 judge。

\subsection{R1、Kimi 和 Qwen 说明了什么}
课程在 case study 里反复对比 DeepSeek R1、Kimi K1.5 和 Qwen 3。它们路径不同，但都指向同一个事实：reasoning RL 的关键不只是“用了 RL”，而是奖励足够可验证、SFT 初始化足够好、难度筛选足够严格，以及 RL 基础设施能支撑长 CoT 训练。

R1 说明了 GRPO 风格 reasoning RL 可以把系统推到极强水平；Kimi 展示了不同 policy gradient 变体和长度控制策略；Qwen 则说明即便数据量没有那么大，只要筛选和训练流程够好，RLVR 仍然能非常有效。课程因此把“GPT-3.5 时代的 RLHF”推进到了“o1 / R1 时代的 reasoning RL”。

\subsection{DeepSeek R1：为什么它成了这一波 RLVR 的中心案例}
官方后半段对 R1 的拆解其实相当具体。第一，R1-zero 说明仅靠 reasoning RL 加上极简的可验证奖励，就已经能得到相当强的性能；第二，真正的 R1 又在此基础上加回了 SFT 初始化、语言一致性奖励，以及后续的常规 post-training。

课程尤其强调两点：一是 `SFT initialization` 对 reasoning RL 的启动阶段非常重要，即便只有少量高质量长 CoT 样本，也能大幅改善后续 RL 可训练性；二是 R1 并没有靠复杂的 process supervision 或 MCTS 才成功，这直接改变了大家对 reasoning RL 门槛的认知。

\subsection{Kimi K1.5：另一条同时期路线说明 objective 还在快速演化}
Kimi 的价值在于它不是简单重复 R1。官方材料说明，它同样依赖难度筛选、长 CoT SFT 和 RL，但在具体损失、长度控制和 curriculum 上给了另一套设计。特别是长度控制部分，Kimi 会在后期显式鼓励更短的正确链条，说明 reasoning RL 的一个真实工程目标，不只是“答对”，还包括“答对且不要太啰嗦”。

这让课程的 message 更完整了：GRPO 打开了门，但真正的工业路线并不会只用一个原始 objective 原封不动地跑到底，而是会围绕数据难度、长度、语言一致性和 reward 设计不断叠 patch。

\subsection{Qwen 3：低数据 RLVR 说明筛选质量可以替代纯数据量}
Qwen 3 的 case study 则把另一个信号拉了出来：reasoning RL 不一定非要海量 RL 数据。官方摘要里明确提到，他们通过 best-of-n 筛难题、剔除无需 CoT 就能做对的样本、手工检查 CoT 质量，并最终只在大约四千级别的例子上做 GRPO。这说明当 reward 足够硬、样本筛选足够狠时，小数据 RLVR 也可能有很高杠杆。

\begin{importantbox}{RLVR 的真实含义}
RLVR 不是“换了一个缩写”。它意味着强化学习第一次在一类大模型任务里拿到了足够硬、足够便宜、足够可扩展的奖励信号。
\end{importantbox}

\subsection{本章小结}
RLVR 之所以重要，不在于它看起来更潮，而在于它让强化学习的优化目标变得更可信。这才是 reasoning 模型近两年突飞猛进的真正背景。

\section{总结与延伸}
第 16 讲把对齐从偏好优化推进到了更结构化的 RL 视角。

第一，\textbf{DPO 之所以流行，是因为它大幅降低了偏好优化门槛。} 但它并不适合所有反馈形式。

第二，\textbf{RLHF 的核心难题不是“会不会实现”，而是目标本身太容易被过优化。} 过优化和模式坍缩几乎是内生风险。

第三，\textbf{PPO 仍然是理解语言模型 RL 的参考系。} 它表达力强，但系统复杂度高、value model 成本大。

第四，\textbf{GRPO 通过组内归一化大幅简化了在线 RL。} 它牺牲了一些理论洁净性，换来了 reasoning RL 的可用性。

第五，\textbf{RLVR 的突破来自奖励变硬。} 当奖励足够可验证时，RL 才真正有机会把模型往更强推理能力上推。

第六，\textbf{R1、Kimi、Qwen 说明 reasoning RL 已经不再只是实验室玩具。} 它是一整套奖励设计、数据筛选、SFT 初始化和 RL 系统工程共同作用的结果。

\begin{warningbox}{我对第 16 讲的压缩提醒}
如果只把这讲理解成“介绍了 PPO、DPO 和 GRPO”，那就漏掉了真正的转向。更重要的结论是：\textbf{一旦偏好奖励开始限制我们继续 scale，能够提供可验证正确性的奖励信号就会成为新的增长引擎。}
\end{warningbox}
